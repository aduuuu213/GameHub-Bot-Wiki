# Wiki文档编写指南

## 文档结构说明

```

├── 09-setting/                 # 系统管理模块 (4个文档) 🔒

```

## 编写规范

### 文档结构模板

每个功能文档应包含以下章节：

```markdown
# [功能名称]

🔒 **需要权限：** [如需要特定权限，在此说明]

## 功能简介
[2-3句话说明该功能的作用]

## 页面说明
### 界面布局
[配图：功能页面截图]
### 主要功能区
[说明页面各区域功能]

## 操作步骤
### [操作1]
1. 步骤1
2. 步骤2
3. ...

## 注意事项
⚠️ [重要提示]
💡 [使用技巧]

## 常见问题
[针对该功能的常见问题]

---
*相关文档：*
- [相关文档1](相对路径)
- [相关文档2](相对路径)
```

### 图片规范

1. **图片存放位置**: `docs/wiki/images/`
2. **命名格式**: `{模块}-{功能}-{序号}.png`
   - 例如: `task-batch-01.png`, `orders-list-01.png`
3. **图片引用**: `![描述](../images/图片名.png)`
4. **图片要求**:
   - 格式: PNG或JPG
   - 清晰度: 建议1920x1080或适当分辨率
   - 文件大小: 单张不超过500KB

### 路径引用规范

#### 相对路径引用
- 同文件夹内: `filename.md`
- 上级文件夹: `../folder/filename.md`
- 跨文件夹: `../other-folder/filename.md`

#### 示例路径
```markdown
# 从 02-task-control/task-batch.md 引用其他文档
- [任务控制总览](overview.md)                    # 同文件夹
- [商品管理总览](../03-archives/overview.md)     # 其他模块
- [快速入门指南](../01-getting-started/quick-start.md)  # 跨文件夹
```

### 内容编写建议

#### 1. 语言风格
- 使用简洁、清晰的语言
- 用第二人称"您"称呼用户
- 避免技术术语，必要时在术语表中解释

#### 2. 操作步骤
- 使用有序列表
- 每步说明要具体、可执行
- 关键操作配截图标注

#### 3. 注意事项
- 使用图标标识：⚠️ (警告)、💡 (技巧)、ℹ️ (说明)
- 突出重要信息
- 说明潜在风险

#### 4. 交叉引用
- 在文档底部添加相关文档链接
- 在正文中适当添加内链
- 保持链接准确性

### 文档更新流程

1. **编写内容**: 根据功能实际情况填充"[待填写]"部分
2. **添加截图**: 截取相关页面并按规范命名
3. **交叉检查**: 确保相关文档之间链接正确
4. **审校**: 检查文字、格式、链接
5. **更新时间**: 修改文档底部的更新时间

- 提供数据解读方法
- 给出分析建议
